# 🚀 Деплой на Vercel

## Пошаговая инструкция для деплоя на https://vercel.com/

### 1. Подготовка проекта ✅
- [x] Build скрипты настроены для Prisma и Next.js
- [x] Fallback режим для работы без базы данных
- [x] Vercel.json конфигурация создана
- [x] Environment variables документированы

### 2. Деплой через веб-интерфейс Vercel

1. **Зайдите на https://vercel.com/**
2. **Нажмите "Add New..." → "Project"**
3. **Подключите GitHub репозиторий:**
   - Выберите репозиторий: `MichaelRogalskiy/manager-rating-system`
4. **Configure Project:**
   - Framework Preset: **Next.js** (автоматически определится)
   - Root Directory: **/** (корень проекта)
   - Build Command: `npm run build` (автоматически)
   - Output Directory: `.next` (автоматически)
   - Install Command: `npm install` (автоматически)

### 3. Добавление Vercel Postgres Database

1. **После деплоя зайдите в проект на Vercel**
2. **Перейдите в Storage tab**
3. **Нажмите "Create Database"**
4. **Выберите "Postgres"**
5. **Настройте базу данных:**
   - Database Name: `elomenedger-db`
   - Region: `Frankfurt (fra1)` (ближайший к Украине)
6. **Vercel автоматически создаст и подключит `DATABASE_URL`**

### 4. Настройка Environment Variables

В настройках проекта на Vercel добавьте:

```
DATABASE_URL=postgresql://... (автоматически добавляется Vercel Postgres)
```

### 5. Миграции базы данных

После подключения базы данных:
1. **Vercel автоматически запустит миграции** через `prisma migrate deploy` в build скрипте
2. **Если нужно запустить миграции вручную:**
   - Зайдите в Functions tab в Vercel
   - Найдите последний деплой
   - Проверьте логи сборки

### 6. Тестирование

После деплоя проверьте:
- ✅ Приложение загружается
- ✅ Создание сессии работает
- ✅ Сравнения менеджеров работают  
- ✅ Рейтинги отображаются
- ✅ База данных подключена (если есть данные)
- ✅ Fallback режим работает (если база недоступна)

### 7. Полезные команды для отладки

```bash
# Локальное тестирование production build
npm run build
npm run start

# Проверка миграций локально
npm run db:migrate

# Push схемы в базу без миграций (для быстрого тестирования)
npm run db:push
```
